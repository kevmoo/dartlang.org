import 'dart:async';
import 'dart:math' show Random, pow;

main() async {
 var count = 10000000;
 print('Compute π using the Monte Carlo method with '
       '$count samples.');
 var pi = await computePi(count);
 print("π ≅ $pi");
}

Future computePi(int samples, {int seed}) async {
 var points = generateRandomPoints(seed).take(samples);
 var pointsInside = points.where((p) => p.isInsideUnitCircle);
 return pointsInside.length / samples * 4;
}

Iterable generateRandomPoints([int seed]) sync* {
 var random = new Random(seed);
 while (true) {
   yield new Point(random.nextDouble(), random.nextDouble());
 }
}

class Point {
 final double x, y;
 Point(this.x, this.y);
 bool get isInsideUnitCircle => pow(x, 2) + pow(y, 2) <= 1;
}

Core Goals

Dart is an ambitious, long-term project but it does limit its scope. It can’t be the answer to all world’s problems (sorry). These are the core goals that drive our design decisions:

Make common programming tasks easy
Application programming comes with a set of common problems (like deep asynchrony) and common errors (like null pointer exceptions). Dart is designed to make those problems easier to tackle, and errors easier to catch.
Provide a solid foundation of libs and tools
A programming language is nothing without its core libraries and tools. Dart’s have been powering very large apps for years now.
Be the stable, pragmatic solution for real apps
Dart may seem ‘boring’ to some. We prefer the term ‘productive’.
No gotchas
There should be a direct correspondence between what you type and what's going to happen.
